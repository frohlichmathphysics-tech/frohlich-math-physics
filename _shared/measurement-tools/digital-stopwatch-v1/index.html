Digital Stopwatch — Test Page

physics/labs/measurement-tools/digital-stopwatch
What to test:
  • Drag the stopwatch body anywhere inside the white frame. Buttons and the lap strip do not initiate a drag.
  • Click Start. Digits count up live every animation frame. The Start button morphs into a green Lap button.
  • Click Lap several times while the timer runs. A cumulative-time chip appears in the strip below the buttons for each press. The strip scrolls horizontally as chips accumulate. The main readout keeps showing total elapsed — splits are not displayed (students compute them).
  • Watch the counter on the right edge of the lap row: it shows N / 20 at all times. When it reaches 20 / 20 the counter turns red — the cap is visible before the next press.
  • Press Lap a 21st time. The counter pulses, the button flashes red for ~150 ms, and the press is not recorded.
  • Click Stop. Digits freeze at the final reading.
  • Click Start again. With laps recorded, Start is disabled — Reset is required first. This is the data-protection guard.
  • Click Reset. Digits return to 00.00, the lap strip clears.
  • API: getCurrentMs(), getElapsedMs(), getLapsMs(), getSplitsMs(). Open the browser console and call them on the global sw instance.
  • Open this page on a phone. Drag and buttons should work via touch; the lap strip should scroll horizontally with a finger swipe.
