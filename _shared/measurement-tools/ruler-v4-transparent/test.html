Ruler — Test Page (v4 transparent, rotating)

_shared/measurement-tools/ruler-v4-transparent · identical API to ruler-v4 wooden · vendor either as ruler.js per applet
What to test:
  • Visibility on cream: the transparent body should be faintly visible against the cream frame background (frosted-white fill, dark gray edge). If it disappears or is too subtle, flag for fill-opacity bump.
  • API interchangeability: all constructor params identical to wooden v4. Swapping ./ruler.js from wooden to transparent (or vice versa) should be the ONLY change needed in an applet.
  • Case 1 (v3 regression): no rotatable flag → behaves identical to v3 (whole body grab, sliver-visible bounds).
  • Case 2 (rotatable, horizontal start):
    • Drag the middle ~80% to translate (cursor: move).
    • Drag the first or last ~50 px to rotate around the OPPOSITE end (cursor: rotation arrow).
    • Hold Shift while rotating → angle snaps to 15° multiples.
    • Pivot dots: dark metallic (not brass), top of each end. Should be visible enough to act as a discoverability cue.
  • Case 3 (rotatable, 90° initial): appears vertical with 0 at top, 100 at bottom.
  • Edge cases: rotate past 180° (numerals upside-down is correct). Drag off-frame is intentionally allowed in rotatable mode.
